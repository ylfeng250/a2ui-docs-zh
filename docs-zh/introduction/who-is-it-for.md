# A2UI 适合谁？

面向构建带有丰富、交互式 UI 的 AI 智能体的开发者。

## 三类受众

### 1. 宿主应用开发者（前端）

构建多智能体平台、企业助手或跨平台应用，其中智能体生成 UI。

**为什么选择 A2UI：**

- 品牌控制：客户端拥有样式和设计系统。
- 多智能体：支持本地、远程和第三方智能体。
- 安全：声明式数据，无代码执行。
- 跨平台：Web、移动端、桌面端。
- 互操作：开源，多种渲染器使用同一规格。

**开始使用：** [客户端设置](../guides/client-setup.md) | [主题定制](../guides/theming.md) | [定义你自己的目录](../guides/defining-your-own-catalog.md)

### 2. 智能体开发者（后端/AI）

构建能生成表单、仪表板和交互式工作流的智能体。

**为什么选择 A2UI：**

- LLM 友好：扁平结构，易于增量生成。
- 丰富的交互：超越文本（表单、表格、可视化）。
- 生成而非工具：UI 作为智能体生成输出的一部分。
- 可移植：一个智能体响应适用于所有 A2UI 客户端。
- 可流式传输：在生成时渐进式渲染。

**开始使用：** [智能体开发](../guides/agent-development.md)

### 3. 平台构建者（SDK 创作者）

构建智能体编排平台、框架或 UI 集成。

你是否将远程智能体引入你的应用？

你是否将你的智能体部署到你不一定控制的其他应用中？

**为什么选择 A2UI：**

- 标准协议：与 A2A 和其他传输 (Transport) 互操作。
- 可扩展：自定义组件目录 (Catalog)。
- 开源（Apache 2.0）。

**开始使用：** [社区](../ecosystem/community.md) | [路线图](../roadmap.md)

## 何时使用 A2UI

在以下场景中使用 A2UI：
- **智能体生成的 UI**：核心用途。
- **多智能体系统**：跨信任边界的标准协议。
- **跨平台应用**：一个智能体，多种渲染器（Web/移动端/桌面端）。
- **安全关键**：声明式数据，无代码执行。
- **品牌一致性**：客户端控制样式。

不要使用 A2UI：
- **静态网站**：使用 HTML/CSS。
- **简单的纯文本聊天**：使用 Markdown。
- **未与客户端集成的远程组件**：使用 iframe，如 [MCP Apps](../introduction/agent-ui-ecosystem.md)。
- **快速构建的 UI + 智能体应用**：使用 [AG UI / CopilotKit](../introduction/agent-ui-ecosystem.md)。

有关详细比较，请参阅 [智能体 UI 生态系统](../introduction/agent-ui-ecosystem.md)。
