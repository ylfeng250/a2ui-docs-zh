# 定义你自己的目录

虽然 [基础目录（Basic Catalog）](../../specification/v0_9/json/basic_catalog.json) 对于启动和引导应用程序很有用，但大多数生产应用都会定义自己的目录以反映其特定的设计系统。

通过定义你自己的目录，你可以限制智能体使用恰好存在于你的应用中的组件和视觉语言，而不是通用输入或按钮。

## 为什么要定义你自己的目录？

每个 A2UI 表面（Surface）都由一个**目录（Catalog）**驱动。目录本质上是一个 JSON Schema 文件，它告诉智能体哪些组件、函数和主题可供其使用。

定义你自己的目录提供以下好处：
-   **设计系统对齐**：限制智能体使用恰好存在于你的应用中的组件和视觉语言。
-   **安全和类型安全**：你将整个目录注册到你的客户端应用，确保仅渲染受信任的组件。
-   **无需映射器**：建议构建直接反映客户端设计系统的目录，而不是尝试通过适配器将通用目录（如基础目录）映射到它。

基础目录只是一个示例，故意保持精简，以便不同的渲染器轻松实现。

## 工作原理

1.  **定义目录**：创建目录定义（JSON Schema），列出你的应用支持的组件、函数和样式。
2.  **注册目录**：将目录及其对应的组件实现（渲染器）注册到你的客户端应用。
3.  **通告支持**：客户端告知智能体其支持的目录（通过 `supportedCatalogIds`）。
4.  **智能体选择目录**：智能体为给定 UI 表面选择一个目录（通过创建消息中的 `catalogId`，如 `createSurface`）。
5.  **智能体生成 UI**：智能体使用该目录中定义的组件按名称生成组件消息。

## 实现指南

建议创建直接映射到现有组件库的目录。

=== "Web (Lit / Angular / React)"

    要在 Web 上实现你自己的目录：
    -   创建包含你的组件定义的 JSON Schema。
- 在你选择的 Web 渲染器中创建自己的 `Component` 对象和 `Catalog` 对象。
    -   将模式或引用 ID 提供给智能体。

    *每个框架的详细指南即将推出。*

=== "Flutter"

    要在 Flutter 中实现你自己的目录：
    -   定义描述你的小部件属性的 JSON Schema。
    -   使用自定义渲染器将模式映射到 Flutter 小部件。

    *详细的 Flutter 集成指南即将推出。*

## 安全注意事项

在定义和注册目录时：

1.  **白名单组件**：仅在你的目录定义中注册你信任的组件。除非严格受控，否则不要暴露提供危险能力（例如执行任意脚本）的组件。
2.  **验证属性**：始终验证来自智能体消息的组件属性，确保它们符合预期的类型约束。
3.  **清理文本**：避免渲染智能体提供的未经清理的内容，除非建立了安全边界。
